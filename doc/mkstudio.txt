1) Дополнительные типы данных для отображения
- double (Вещественный двойной точности)
- Восьмиричный
- Символ CP1251 (Win) (только для байтового типа)
- Символ CP866 (DOS)  (только для байтового типа)
- Символ KOI8-R       (только для байтового типа)
- Если возможно, сделать отображение строки (терминирующий 0) (только для байтового типа)
- Символ UNICODE      (UTF-8, UTF-16 для байтового и словного типов соотв.)

2) Дополнительные протоколы
- Классический MODBUS RTU:
       Адреса регистров  Тип  Команда Чтения        Команды записи
 DOut  00000.. 9999       0   COIL_STATUS(1)        SINGLE_COIL(5), MULTIPLE_COILS(15)
 DInp  10000..19999       1   INPUT_STATUS(2)
 AOut  40000..49999       4   HOLDING_REGISTERS(3)  SINGLE_REGISTER(6), MULTIPLE_REGISTERS(16)
 AInp  30000..39999       3   INPUT_REGISTERS(4)
 Адрес регистра:
 - ведущий символ - тип регистра
 - 4 знака 0..9999 - номер регистра
 Пересчет номера регистра в адрес:
 - аналоговый - равен адресу в памяти MODBUS MIKKON
 - дискретный - деленый на 8 равен адресу в памяти MODBUS MIKKON
 ВНИМАНИЕ !!!
 - Один аналоговый регистр - ДВА байта.
 - Один дискретный регистр - ОДИН бит.
 - FIX !!! Для дискретных регистров номер должен быть кратен 8, поскольку
   во всех прошивках битовые операции для чтения не сделаны.
 - Запись одного дискретного регистра сделана правильно
 - FIX !!! Запись многих дискретных регистров также работает только для номеров,
   кратных 8.
- MODBUS ASCII (классический и MIKKON) - ТОЛЬКО по запросам трудящихся
